## What NOT to Do (Lessons Learned)

1. **Do NOT do full rewrites of views.tsx** — the user prefers incremental, targeted changes
2. **Do NOT turn the portal into a landing page** — the reference image was for quality inspiration, not literal layout copying
3. **Do NOT remove the established layout patterns** — sidebar+main+rail, card-based UI, sticky header are all working
4. **Do NOT make orgSlug required in views that previously had it optional** — this breaks page files
5. **Do NOT add non-existent properties from ContactConfig** — `businessEmail`, `phone`, `businessAddress` do not exist
6. **Do NOT change the max-width or fundamental grid without explicit user request**
7. **Do NOT make dark/charcoal the primary CTA color** — the user likes the current accent-driven CTAs
8. **Do NOT add atmospheric gradient orbs or dramatic background shapes** — rejected by user
