### 2. Sequences Page Ownership Bug Fix
**Root Cause:** `getSequenceSettings` server action's `canEdit` check returns `false` for actual owners when `userOrgPreference` is out of sync.

**Fix:** Client-side fallback to `activeOrg?.role === "owner"`:
```typescript
const isOwner =
  canEditSettings === true ||
  activeOrg?.role === "owner" ||
  activeOrg?.role === "OWNER";
```

**Commit:** `196bfa40` — fix(settings/sequences): use client-side org role as fallback for isOwner

### 3. Usage Dashboard BigInt Fix
**Root Cause:** Prisma 5.x `.count()` returns `bigint`. The usage dashboard `pct()` function crashed with "Cannot mix BigInt and other types" when doing `current / limit`.

**Fix (3-layer defense):**
1. `src/lib/usage-metering.ts` — `Number()` all `.count()` results + `normalizeBigInts()` for DB reads
2. `src/app/app/settings/billing/usage/page.tsx` — `toNumber()` helper for plan limits
3. `src/features/settings/components/usage-dashboard-client.tsx` — hardened `toNumber()` utility

**Commit:** `8e963b7e` — fix(usage): normalize BigInt values from Prisma .count() to prevent JS runtime crash

### Usage Dashboard Redesign (PR #288)
**Branch:** `feature/platform-rebrand-redesign-usage-dashboard`
**Target:** `feature/platform-rebrand-redesign`
**PR:** https://github.com/Fenar7/PaySlip_Generator/pull/288

**Problem:** Old UI was a dense vertical list of 10 progress bars. Users couldn't easily scan or understand limits at a glance.

**Solution:** Card-based layout with icons, logical grouping, and summary stats.

**New Design:**
- **Summary stat row:** Total Limits | Near Limit | Unlimited | Active Metrics
- **Alert badge:** Shows when any limit is ≥70% capacity (red, with warning icon)
- **Grouped metric cards:**
  - 📄 **Documents** — Invoices, Quotes, Vouchers, Salary Slips
  - 👥 **Team & Access** — Team Members, Active Portal Sessions
  - 💾 **Storage & Media** — Storage Used, Pixel Jobs Saved
  - ⚙️ **Platform** — Webhook Calls, Active Share Bundles
- **Each card has:**
  - Icon in color-coded circle (green/amber/red background)
  - Label + current/limit numbers (large bold)
  - Percentage badge (or "Unlimited" pill)
  - Thin animated progress bar
  - Warning text when near/at limit

**Files Changed:**
- `src/features/settings/components/usage-dashboard-client.tsx` — complete rewrite (~319 lines)
- `src/app/app/settings/billing/usage/page.tsx` — removed inner title, layout cleanup

**Key Implementation Details:**
- Uses `GROUP_ORDER` array to control section display order
- `ICONS` and `GROUP_ICONS` maps define icons per metric and per group
- `statusColor()` returns `{ bg, text, bar }` based on percentage
- `MetricCard` sub-component encapsulates card rendering
- BigInt-safe via `toNumber()` utility (accepts `bigint | number | string | null | undefined`)
